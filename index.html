Que son los roles en un equipo TI

Los roles en un equipo TI determinan como va a ser el funcionamiento de la compañia, que se busca en el desarrollo y a su vez determinan las metodolgias que seran usadas a la hora de desarrollar algun software

Caracteristicas y composicion

existen varios tipos de roles, vamos a ver los mas comunes.

  • Cliente: El cliente se encarga de definir revisar y autorizar los requerimientos para la construcción del software
  • Lider de proyectos: se encargara de tener entrevistas con el cliente para poder recopilar la informacion necesaria para solucionar el problema, ademas controla los avances del proyecto
  • Desarrollador de software: se encargara de llevar dicha informacion a codigo y documentacion y mediante los requerimientos levantados desarrollar un software que cumpla con las espetivas del cliente

Entre otros roles podemos encontrar:


  • Gerente del proyecto
  • Analista de sistemas
  • Diseñador
  • Responsable de calidad y pruebas

conclusiones

Cada equipo de desarrollo requiere union para poder resolver el problema del usuario, siempre se busca la satisfaccion antes que un desarrollo rapido que pueda resultar en problemas

Conclusiones

dentro del area del desarrollo hay muchos mas roles, los cuales son igual de importantes a los demas en el desarrollo de software, cada especializcion atiende a un problema especifico
